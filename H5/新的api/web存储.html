优势和劣势

  • 拓展了cookie的4k限制
  • 会将第一次请求的数据直接存储到本地，这个相当于5M大小的针对于前端页面的数据库。
  • 只能支持string的存储,不能json需要转换
  • 不能被爬虫抓取
  • 本质上是对字符串的读取，如果存储内容多的话会消耗内存空间，导致页面变卡

注意事项

  • 三种写入方法
  • 读取也是三种方法
  • 也是遵循同源策略，不同网站不能共用
  • 删除消息
  • 适用范围：localstorage可以存储一些不需要和服务器交互的数据，比如导航栏的状态，一些普通的数据进行缓存。甚至我们可以存储html片段，js或者css文件段。可以控制来存储一些不常改动的js/css文件。减少用户请求宽带的同时优化整个页面的加载速度。
  • h5游戏通常会产生一些本地数据，localstorage非常适合。如果遇到一些内容特别多的表单，为了优化体验，我们可以把表单拆分为多个子页面，然后按照步骤引导用户填写。
  • 使用场景
    • 缓存一般的信息。如搜索页触发的城市，到达城市，非及时定位信息。
    • 缓存城市列表数据
    • 你可以把你想要的任何文本数据放在那里，然后从同一域的任何页面上取回。
