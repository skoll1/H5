对于网址可寻的资源-Cache API

对于其他的所有数据，使用indexDB

其他不推荐的存储机制

  • web storage 同步的，不支持网页工作线程，并对大小和类型（仅字符串）进行限制。
  • cookie:具有自身的用途，并且也会大小进行限制。
  • web sql:不具有广泛的浏览器支持，因此不建议使用

如何了解我的应用当前使用了多少存储空间

  • Quota Management API
  • Storage Quota Estimate API

缓存逐出是如何工作的

  • chrome 耗尽空间的时候采用LRU策略
  • firefox:在整个磁盘装满的时候采取LRU策略
  • safair:无逐出
  • edge:无逐出
  • 策略：“尽力而为”：在不干扰用户的情况下清除存储，但是对于长期或者关键数据而言持久性较差 indexDB，cache api
  • 策略：“持久化”：存储在存储容量降低时不会自动清除，需要用户手动清除。

可以使用的资源

  • chrome <6%
  • firefox <10%
  • sfair <50mb
  • IE 10 <250mb
